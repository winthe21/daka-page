为什么GitHub Pages显示代码而不是网页？

常见原因

当GitHub Pages显示HTML源代码而不是渲染的网页时，通常是由以下原因造成的：

✓ 文件没有命名为index.html（GitHub Pages的默认文件）
✓ 文件没有放在仓库的根目录
✓ GitHub Pages设置不正确
✓ URL访问方式错误
✓ HTML文件语法错误

如何判断问题所在

要确定问题的具体原因，请检查以下几点：

  1. 访问的URL是否正确（应该是https://用户名.github.io/仓库名/）
  2. GitHub仓库中的文件结构
  3. GitHub Pages的设置（Settings → Pages）
  4. HTML文件是否包含正确的DOCTYPE声明

解决GitHub Pages显示代码的问题

方法一：重命名文件为index.html

GitHub Pages默认寻找名为index.html的文件作为主页。

# 将您的HTML文件重命名为index.html
mv check-in-page.html index.html

然后重新上传到GitHub仓库的根目录。

方法二：正确配置GitHub Pages

  1. 进入您的GitHub仓库
  2. 点击"Settings"选项卡
  3. 在左侧菜单中选择"Pages"
  4. 在"Source"部分，选择您的分支（通常是main或master）
  5. 选择文件夹为"/(root)"
  6. 点击"Save"

方法三：使用正确的URL访问

如果您的文件不是index.html，您需要使用完整的URL路径：

# 如果文件名为mypage.html
https://用户名.github.io/仓库名/mypage.html

注意URL是区分大小写的，请确保大小写匹配。

预防GitHub Pages问题的措施

最佳实践

✓ 始终将主页面文件命名为index.html
✓ 将HTML文件放在仓库的根目录
✓ 确保HTML文件包含正确的DOCTYPE声明
✓ 使用相对路径引用CSS、JS和图像文件
✓ 在推送更改后等待几分钟让GitHub Pages构建完成

重要提示

GitHub Pages构建可能需要几分钟时间。更改设置或上传新文件后，请耐心等待1-5分钟，然后刷新页面查看结果。

快速修复指南

五分钟修复步骤

  1. 将您的HTML文件重命名为index.html
  2. 确保文件在仓库的根目录（不在任何文件夹内）
  3. 上传到GitHub仓库
  4. 进入仓库Settings → Pages
  5. 选择分支（main或master）和根目录
  6. 保存设置并等待2分钟
  7. 访问https://用户名.github.io/仓库名/

检查清单

如果问题仍然存在，请检查以下各项：

✓ HTML文件以<!DOCTYPE html>开头
✓ GitHub Pages设置正确（分支和根目录）
✓ URL中没有拼写错误
✓ 等待了足够的时间让GitHub构建页面
